# 规划提速与编排画布优化计划

## 目标

- 明确任务直接走快速规划，减少不必要的 LLM 调用。
- 复杂任务仍保留员工编排、会议、验收与知识检索能力。
- 编排画布显示中文短标题、执行角色、执行器、耗时与实时日志入口。

## 改动边界

- 不新增依赖。
- 不重写现有规划器、运行器或前端结构。
- 优先通过本地启发式补强 `makePlan`、`api.plan` 和画布节点渲染。

## 实施步骤

1. 先补规划提速和画布字段的红灯测试。
2. 增加简单任务快速规划、本地主部门判断和规划统计。
3. 扩展计划接口，输出中文短标题、角色行、耗时和日志摘要。
4. 优化画布节点显示，并支持点击查看步骤执行详情。
5. 跑定向测试、全量测试和浏览器可用性检查。
