# orch — 轻量 Agent 编排器

发一个任务，自动拆解并分配给 Claude / Codex 协作（并行分工 + 改测循环）。本地单人工具。

## 装 + 跑

```bash
cd orch
npm install
npm start              # 起 http://localhost:3000
```

浏览器开 `http://localhost:3000`，进入 **Maestro 编排工作台**：点右上「新建任务」下发，
任务列表/详情看 Claude→Codex 接力与实时日志，总控台看真实指标与实时编排活动流，
编排画布看真实执行计划。**全部为 orch 真实数据**：agent=真实适配器(Claude/Codex),
部门=按 agent 角色派生(开发/QA),项目=按任务 project 字段聚合,人员=当前操作者。

> 真跑需要本机已登录 `claude` 和 `codex` CLI（`claude -p`、`codex exec` 无头模式）。
> codex 较慢（约 100s/步），属正常。

> ⚠️ **安全**：为让 agent 无人值守自主改文件/跑命令，适配器用了
> `claude --dangerously-skip-permissions` 和 `codex --dangerously-bypass-approvals-and-sandbox`，
> 即**绕过所有权限检查、无沙箱**。这是"自主编排器"的固有代价:派下去的任务能对工作目录做任何事。
> 只在你信任的工作目录/任务上用。

## 怎么运作

- 一份 **plan**（步骤 + 依赖 + 派给谁）是核心。模板和 LLM 都只是生成 plan 的方式。
- 下发任务时选模式：**智能拆解(LLM)** 让 Claude 把目标拆成多步、自动指派可用 agent（能跑出"建站"这类多步流程）；**模板** 走固定 开发→验证→改测循环。
- 引擎拓扑调度：无依赖的步骤并发跑，有依赖的串行，`loop` 步骤至少跑一次 body、失败按 `until`/`max` 重试。
- **并发上限**：全局信号量 `ORCH_CONCURRENCY`(默认 3)限同时运行的 agent 子进程,防压垮单机。
- **取消/停止**：运行中任务详情有「停止」——按 **PID 定向**杀该任务 spawn 的子进程树(绝不按镜像名,不误伤别的 claude),状态转「已取消」。
- **重试失败步骤**：失败任务详情「↻ 重试失败步骤」——已完成步骤不重跑,只重跑失败/未完成的(执行器撞会话限额时,等重置后一键续跑,不浪费已花成本);撞限额会在日志给出明确提示。
- **运行时员工署名**:实时活动流、任务列表行(「▸ 🎨 工程部·前端开发工程师 正在做 fe」)、执行器卡都显示**哪个部门的哪个员工在做哪一步**。
- **员工任务简报**:每步执行时自动注入——总任务全貌、流水线位置(第n/m步、上游谁、产出交给谁)、工作目录现有文件清单,员工带着全局与现场感知干活。
- **成本统计**：claude 用 `--output-format stream-json` 采真实 token/成本;其它 agent 按 `pricing` 估算。总控台「今日成本」、任务详情「成本」。
- **审批门**：新建任务勾「逐步审批」→ 出计划后停在「待审批」,详情页「批准并运行」/「取消」。
- **遇抉择问我(可选)**：新建任务勾「遇抉择问我」→ agent 无合理默认时输出 `NEED_DECISION` 停下,任务转「待输入」,详情页显示问题+输入框;答复后从该步续跑(已完成步跳过)。不勾则默认自主决定不反问。
- **接力记录有意义**：每步展示该 agent 的产出摘要(具体做了啥),不再只显状态。
- **产出预览 + 打开目录**：任务详情「产出物预览」列出产出文件并**直接内嵌预览**——HTML(预览/「查看源码」切换)、**Markdown 渲染**(渲染/源码切换)、图片、视频、以及 js/css/json/md 等**文本源码**;「打开目录」在系统文件管理器打开产出目录。
- **应用广场 + 一键发布**：任务完成(含 html)→ 详情「🚀 发布到应用广场」(**仅管理员**可发布/删除)→ 侧栏「应用广场」出现卡片(实时缩略图),点「运行」在站内全屏 iframe **直接使用**该应用,或新标签打开。
- **继续开发**：已完成/已取消/失败的任务,详情「继续开发」填新需求 → **在原任务上追加新一轮步骤**(不新建任务),复用原产出目录,agent 先读现有文件再扩展(不从零重写);接力记录里能看到历轮步骤(新步骤 id 加前缀防冲突)。
- **登录 + 权限(服务端强制)**：首次进站需登录,默认管理员 `admin / admin`(登录后右下角账号可改密)。任务归属=登录用户;**普通用户只看自己的项目**,项目 owner/管理员可在项目详情「项目授权」把项目授权给其他用户;被授权用户可查看,但**非本人任务只读**(不能停止/审批/继续/发布)。管理员看全部、可管理 Agent/人员/部门。
- **部门管理**：管理员在总控台「+ 新建部门」建部门并勾选归属 Agent;新建/编辑 Agent 时可选所属部门。
- **部门员工(角色)体系**：内置 **20 个部门 / 87 名 AI 员工**(源自 [agency-agents-zh](https://github.com/jnMetaCode/agency-agents-zh) 专家角色库,原文压缩为中文角色卡):工程/设计/产品/测试/项目管理/营销/销售/安全/金融/法务/HR/支持/战略/供应链/游戏开发/专项/付费媒体/学术/GIS/空间计算。每名员工=角色提示词(身份/关键规则/交付物标准)+绑定执行器(默认 claude,测试部 codex)。
  - **任务默认按部门员工拆分**:编排器读「部门→员工」目录,把任务派给最合适的员工(如 落地页→工程部·前端开发工程师,种草笔记→营销部·小红书运营师);执行时员工角色提示词自动注入。
  - 部门详情=员工卡片墙;接力记录/画布节点显示「部门·员工」署名+emoji;管理员可在部门页「+ 新增员工」/移除员工。
  - 例外:新建任务显式只勾一个执行器且无文字编排 → 该执行器单步直做(不走员工拆分)。种子文件 `roles-seed.json`(v2,升级版本号会覆盖内置员工卡,用户自建员工不动)。
- **总调度(最高权限)**:用户直接建任务时由「总调度」负责拆解/调度/分配——可跨所有部门所有员工,知晓各部门标准流程与质量门方法论(顺序交接、质量门禁通过才推进、失败退回重做,参考 agents-orchestrator)。
- **部门标准作业流程(可编辑)**:每部门内置标准流程(如工程部:快速原型→前端→后端→AI(可选)→代码评审(🚦质量门)→技术文档);部门详情可**编辑**——排序/增删环节、切换必经/可选、设质量门。部门任务按流程拆分,可选环节由总调度按需取舍,质量门用 loop 包住"产出→门禁",FAIL 重做(最多3次)。
- **部门执行器池**:部门详情可勾选该部门允许的执行器(claude/codex/gemini…),**该部门任务只能用池内执行器**(不勾=不限)。
- **下发部门任务**:部门详情「下发部门任务」直接在该部门建任务(只用本部门员工,按其标准流程)。
- **员工深度智能化**:87 名员工为结构化角色卡(平均700字):【身份】【关键规则】【工作流程(分步方法论)】【交付物标准】【交接】(结尾必写交接备忘:产出清单/关键决策/下游须知);评审/核查类员工额外有【判定】(明确 PASS/FAIL+问题清单)。引擎自动把上游交接备忘注入下游步骤(多依赖全部合并)。
- **编排画布**:节点图渲染当前活动任务的**真实计划**——「任务下发」起点 → 各步骤节点,**连线=依赖**、连线上有**流动光点动画**展示流转、运行中的边为流动虚线,颜色=真实状态(排队/运行/完成/失败)。
- **事件溯源**:每任务状态变迁落 `events` 表(历史/审计)。
- **产出物归档**：任务在 `运行目录/data/<用户>/<项目>/<任务>/` 里干活,产出文件落这;不同任务天然隔离。
- **隔离(可选)**:新建任务可选 `worktree`(git 仓内每任务独立分支目录)或 `container`(仅对设了镜像的自定义 agent 走 `docker run`,claude/codex 因依赖宿主登录始终本地)。
- 同一任务各步骤共享该目录,顺序产物天然可见。多任务并发跑,同一 agent 可跨任务/步骤并发——在 **Agent 详情页的实时控制台**看它所有并发活动,每行标 `[T任务·步骤]`。

## 新建 / 配置（界面里点）

- **新建任务**：右上「新建任务」或画布下发按钮 → 填目标、选项目、选模式 → 下发。
  - **选用哪些 Agent**：勾选只用指定 agent(不选=自动/全部)。只选一个 + 无编排 → 该 agent 单步直做。
  - **文字编排**：填「claude 开发、codex 测试」这类,严格按你的编排分步指派(约束在所选 agent 内)。
  - **智能细化需求(默认开)**：下发前先由 LLM 把简短需求扩写成高质量 brief(目标/功能/结构/选型/验收)再交给 agent,产出更好。
- **Agent 团队**：
  - **CLI 智能体自动发现**：启动扫描本机已装的 `claude / codex / hermes / gemini / aider / qwen / cursor-agent`，存在即自动加入，**无需手动添加**。
  - **手动新增模型**：「+ 新建 Agent」选类型——CLI 智能体 / 大模型(DeepSeek 等) / 视频模型(Seedance 等) / 语音模型。CLI 类必填命令；其余类型命令可留空（登记为团队成员，执行接入为后续能力，可先填命令包装）。
  - 进 agent 详情可「编辑 / 删除」（管理员）。
- **新建项目**：项目页「+ 新建项目」；任务下发时归属到项目。
- **新建人员 + 分配**：人员页「+ 新建人员」勾选 agent；或某人「分配」改归属。
- **当前身份**（无登录）：点左下角用户可切换身份；选中某人后 Agent 团队只显示其被分配的 agent（操作者/无分配看全部）。
- **全局搜索**：顶栏搜索框输入关键词回车，跨 任务/Agent/项目/人员 出结果页。

## 结构

```
server.js     入口:express + ws,起服务,真实活动流广播
api.js        把真实 tasks/steps/logs 派生成 Maestro 前端数据(agents/部门/项目/人员)
runner.js     串起 出plan→执行→落库,事件实时广播
planner.js    模板匹配 / 调 LLM 出 plan
engine.js     调度 + 循环(核心)
workspace.js  工作区(共享目录)
store.js      SQLite 单文件
adapters/     generic.js(数据驱动,按 agent 定义跑) + cli.js(spawn 封装) + echo(测试)
templates/    预设工作流 yaml
web/          Maestro 前端:index.html(模板) + runtime.js(迷你dc运行时) + app.js(逻辑+真接线)
```
